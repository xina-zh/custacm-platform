# training-data-codeforces

This module is the Codeforces vertical OJ slice for `platform-training-data`.

It owns Codeforces ODS ingest application code, the Codeforces adapter/facade for admin recent-lookback submission collection, Codeforces ODS/DWD/DWM/DWS schema migrations, idempotent SQL task resources, the SQL task manifest, Codeforces-specific ODS purge and refresh-interval JDBC adapter, local fixtures, and tests. AtCoder ODS DDL, Kenkoooo collection, and ODS purge adapters live in `training-data-atcoder`; AtCoder same-layer DWD/DWM/DWS table DDL lives in `training-data-common`. OJ handle-account app/domain/infra/web code, public DWD/DWM/DWS read-side query services, student-data purge orchestration, shared collection orchestration, OJ collector dispatch, process-local job orchestration, shared SQL task runner wiring, generic warehouse refresh service/handler, and scheduler wiring live in `training-data-common`; this module only contributes Codeforces Spring beans, collection callbacks, refresh interval SQL, and SQL task resources.

## Layer Rules

Dependency direction:

```text
config -> app/domain/infra
infra -> domain
```

Rules:

- `domain` must not depend on `app` or `infra`.
- `app` orchestrates use cases and should not build SQL.
- `infra` owns parsing, JDBC SQL, and database row mapping.
- `config` wires Spring beans and should not contain business logic.
- Codeforces warehouse time fields use UTC+8 (`Asia/Shanghai`) semantics.

## Directory Layout

```text
src/main/java/com/custacm/platform/trainingdata/codeforces/
  app/                 # application use cases; no HTTP or JDBC details
  config/              # typed source properties and Spring Bean wiring
  domain/              # Codeforces ODS, collection, parser, and ODS purge contracts
  infra/               # Codeforces source, parser, JDBC writer, and Codeforces ODS purge adapters

src/main/resources/
  db/migration/        # Flyway migrations owned by this OJ module
  fixtures/codeforces/ # stable local Codeforces test fixtures
  sql/ods/             # ODS upsert SQL
  sql/dwd/             # DWD set-based transform SQL
  sql/dwm/             # DWM set-based transform SQL
  sql/dws/             # DWS set-based transform SQL
  sql/tasks/           # SQL task DAG manifests

src/test/java/com/custacm/platform/trainingdata/codeforces/
  app/                 # collection app tests
  domain/              # Codeforces model validation tests
  infra/               # source client, parser, ODS writer/purge, and warehouse SQL task tests
```

`target/` is generated by Maven and is not part of the maintained source layout.

| Submission collection job | `POST /api/training-data/admin/codeforces/submissions:collect-batch-jobs` plus list endpoint -> `OjSubmissionCollectionJobService` | `studentIdentities`, `lookbackHours`, `refreshWarehouse`, optional `ojName` | `OjSubmissionCollectionDispatcher`, `OjWarehouseRefreshDispatcher`, OJ collection handlers, configured common refresh handlers | ODS, DWD/DWM/DWS through optional refresh | Admin-only in-process batch job for frontend polling; uses the common job service, runs one active job at a time, records requested OJ plus per-identity pending/running/success/failure state, optionally refreshes produced batches for the selected OJ, and retains recent job snapshots in memory. | Student data purge | `DELETE /api/training-data/admin/students/{studentIdentity}/oj-data?ojName=CODEFORCES` -> `OjStudentDataPurgeService.purgeStudentData` | `studentIdentity` plus required `ojName` | `OjOdsDataPurgeRepository.purgeAllByHandle` + `OjWarehouseDataPurgeRepository.purgeAllByHandle` | ODS/DWD/DWM/DWS | Admin-only transactional deletion of the requested OJ's training-data rows. ODS deletion is OJ-specific; DWD/DWM/DWS deletion is common by OJ name and handle. OJ handle-account bindings are kept. Missing bindings return zero deletion counts instead of `404`. |

Null time/date bounds mean no lower or upper time/date bound. Null problem rating bounds mean all rated and unrated rows. Setting either `minProblemRating` or `maxProblemRating` filters to the requested inclusive rating interval and excludes unrated rows. Submission detail HTTP queries use 1-based `page`, default `limit=15`, max `limit=2000`, exact `total`, computed `totalPages`, `hasMore`, and newest-first ordering by submitted UTC+8 time then submission id. App-layer warehouse queries require an OJ handle-account binding for the requested OJ whenever they need to translate between `studentIdentity` and handle.
