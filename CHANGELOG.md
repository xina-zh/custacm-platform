## 未发布

- JSON 失败响应改为真实 HTTP 4xx/5xx，受保护请求只接受规范的 `Bearer <JWT>`；列表分页统一限制 `pageSize <= 100`，调用方不得继续依赖 HTTP 200 失败包、裸 token 或无上限分页。
- 评论根节点按页读取，单次响应内所有当前页根评论合计最多返回 500 条回复，并通过 `repliesTruncated` 标识截断。
- 数据库迁移演进至 V049：V042 增加评论树查询索引，V043–V049 为分类/标签名称、文章标签对及其引用增加唯一约束、索引和外键；旧库升级前必须按部署文档备份并完成重复与悬空引用预检。
- 管理员可逐项设置比赛奖项是否仅登录后可见；游客的赛事列表、赛事详情和公开个人名片不会返回受限奖项，登录用户可查看。公开赛事列表中的团体程序设计天梯赛、百度之星和蓝桥杯仅展示可见奖项计数，进入详情后展示明细。V050 为存量奖项新增该开关且默认保持公开，前后端与迁移需同步发布。
- 比赛创建由必填年份改为可选完整日期，公开页、管理页和个人荣誉优先展示完整日期；年份区间筛选仍按日期派生年份执行，历史比赛保留原年份且不会被伪造为某个具体日期。V051 新增可空日期并允许无日期比赛的筛选年份为空。
- OJ 提交采集改为在同一逻辑批次内按源分页分块写入，accepted 汇总改为数据库按 handle 与 difficulty 聚合，避免跨页提交堆积和按日明细回传；手动与定时采集均允许 Codeforces、AtCoder 并发执行，手动任务仍按同 OJ 互斥。

## 近期里程碑

### 2026-07-15 - 恢复全站双主题

- Blog 与 Training 恢复手动日间/夜间主题；首次访问为日间，用户选择持久化到 `custacm.theme`，并在同源 Training frame 与跨标签页间同步。
- 日间沿用首页与赛事荣誉的暖米色体系，夜间统一采用文章目录的暖黑、米白与陶土橙体系；业务图片、头像和 rating 色阶保持原色语义。

### 2026-07-15 - 完成统一门户与首页编排

- Blog 首页、赛事与个人荣誉、文章回收站和备份能力完成整合；两份 Vue 3 构建共享同一套日间/夜间视觉语义。
- 数据库迁移已演进至 V041，前后端与迁移需要作为一个版本发布。

### 2026-07-13 - 收敛 Blog 单体边界

- 移除退役页面、旧统计与通知链路，账号、OJ handle、文章和训练 HTTP 能力收敛到唯一 Blog API。
- 文章删除改为七天回收站；训练账号存储规范化为 `training_member` 与 `oj_handle_binding`。

### 2026-07-11 - 统一运行拓扑

- Blog API 成为唯一 Spring Boot 后端；一个 Nginx `frontend` 同时托管 Blog `/` 与训练中心 `/training/**`，浏览器 API 统一使用 `/api/**`。
