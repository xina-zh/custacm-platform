## Known TODOs

- `platform-common/common-web` has no source files yet.
- Placeholder backend modules should stay out of the Maven reactor until their first runnable slice is implemented. `frontend` is now a runnable Node/Vite slice and remains outside the Maven reactor.
- Training-data currently implements Codeforces and AtCoder as vertical OJ warehouse modules with ODS, DWD, DWM, and DWS tables, AtCoder startup problem metadata bootstrap, OJ-owned SQL task resources invoked as collection-job refresh callbacks and after enabled scheduled collection when a batch is produced, enabled-by-default OJ-name-aware recent-lookback submission collection scheduling backed by shared collection orchestration, process-local collection job polling backed by common job helpers for frontend batch collection, UTC+8 warehouse time semantics, per-OJ difficulty bucket policies, public Java/HTTP single-user/problem DWD/DWM/DWS query support with backend-paginated newest-first DWD submission details and DWM first-accepted details, admin student-data purge, and a public `studentIdentity` to OJ handles plus per-OJ collection-state map in `oj_handle_account`. Persistent pipeline run state and ADS remain future work.
- Auth has a lightweight `ErrorResponse` for current auth use cases. Cross-module unified error response and request tracing are future work documented in [../logging.md](../logging.md).
