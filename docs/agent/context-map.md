## Known TODOs

- `platform-common/common-web` has no source files yet.
- Placeholder modules should stay out of the Maven reactor until their first runnable slice is implemented.
- Training-data currently implements Codeforces as a vertical OJ warehouse module with ODS, DWD, DWM, and DWS tables, SQL task resources, admin-triggered SQL task DAG refresh, disabled-by-default recent-lookback submission collection scheduling, UTC+8 warehouse time semantics, public Java/HTTP DWD/DWM/DWS query support, and a Codeforces-specific `studentIdentity` to handle binding table used by app-layer personal queries and admin HTTP collection. Persistent pipeline run state, cross-OJ handle accounts, and ADS remain future work.
- Auth has a lightweight `ErrorResponse` for current auth use cases. Cross-module unified error response and request tracing are future work documented in [../logging.md](../logging.md).
